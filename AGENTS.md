# AGENTS.md

## 项目目标

本仓库面向学生、研究生、职场新人和普通用户，记录 AI 工具在学习、研究、求职、办公与内容创作中的实际用法。内容应帮助读者建立方法，而不是制造工具焦虑。

## 新增内容的硬性要求

1. 必须从一个可理解的真实场景出发，说明人物、任务和限制。
2. 必须写出 AI 能帮什么、不能帮什么，以及人工需要承担的判断。
3. 必须包含边界、风险或核查方法。
4. 必须避免广告口吻、夸张承诺和无依据的工具排名。
5. 必须避免鼓励学术不端，不得暗示 AI 可以替代独立学习、研究或考试。
6. 必须优先写给学生和普通用户看，少用行话；专业概念出现时要解释。
7. 必须保持 Markdown 层级清楚，标题从 `#`、`##`、`###` 依次使用。
8. 新增文件后必须更新 README 中相应目录链接。

## 写作风格

- 中文为主，工具名、技术名词可保留英文。
- 使用具体动作和例子，不堆抽象概念。
- 可以写个人观察和不确定性，但不要虚构测试数据或用户故事。
- 少用整齐划一的排比，避免每段都写成“首先、其次、最后”。
- 工具比较应按任务说明差异，不写绝对胜负。
- 提示词必须能被读者替换信息后使用，并说明输入材料和输出限制。

## 事实与时效

- 涉及产品功能、访问条件、政策和价格时，应注明核查日期或避免写死。
- 引用文献、统计和新闻时，应提供可核查来源。
- 不把 AI 生成的引用当作来源。
- 发现过时内容时，优先修订并在提交说明中写清变化。

## 隐私与安全

- 示例使用虚构姓名和去标识化信息。
- 不提交个人证件、联系方式、未公开数据或公司机密。
- 医疗、法律、金融等高风险问题只讨论信息整理方法，并提醒咨询专业人士。

## 提交前检查

- 扫描项目约定禁止出现的营销表达。
- 检查内部 Markdown 链接和文件名。
- 检查文章是否包含场景、方法、例子、错误、边界与小结。
- 检查提示词是否要求 AI 不编造材料。
- 确认 `git diff` 只包含本次任务相关修改。
